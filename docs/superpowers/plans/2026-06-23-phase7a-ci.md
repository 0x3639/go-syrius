# Phase 7a — CI Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a GitHub Actions PR gate (frontend checks, security scanners, and a 3-OS build/test matrix) that enforces the test + security gates CLAUDE.md promises but the repo currently lacks.

**Architecture:** One workflow `.github/workflows/ci.yml` with three parallel jobs (`frontend`, `security`, `build-test` matrix). Two prerequisite code changes make the security gate green: a Go toolchain bump (clears 14 stdlib vulns) and a gosec triage pass (fixes 7 findings, justifies 6). govulncheck runs behind a small allowlist script that hard-fails on any vuln except 5 upstream-blocked go-ethereum p2p IDs.

**Tech Stack:** GitHub Actions, Go 1.25.11, Wails v2.10.1, pnpm 10.17.1, govulncheck, gosec.

## Global Constraints

- **Toolchain:** Go is bumped to `1.25.11` in `go.mod` (Task 1); CI uses `go-version-file: go.mod`. Wails `v2.10.1`. pnpm `10.17.1` (honor `frontend/package.json` `packageManager`).
- **No `GOWORK=off` in CI** — it is a local-only workaround for a parent `go.work`; CI checks out the repo standalone. (Local dry-run commands in this plan still need `GOWORK=off GOTOOLCHAIN=auto` because they run on the developer machine.)
- **Integration tests excluded:** `internal/spike/*_integration_test.go` are behind `//go:build integration`; never pass `-tags integration` in CI.
- **govulncheck allowlist (exactly these 5, no more):** `GO-2026-4314`, `GO-2026-4315`, `GO-2026-4507`, `GO-2026-4508`, `GO-2026-4511` — go-ethereum devp2p/RLPx p2p DoS/handshake vulns, reachable only via the opt-in embedded node, no key/fund exposure, un-bumpable without breaking go-zenon, deferred pending go-zenon's libp2p migration. The gate must hard-fail on any vuln ID NOT in this list.
- **gosec suppressions:** only inline `// #nosec G### -- <justification>`; never blanket-disable a rule globally.
- **Linux WebKit dep for `wails build`:** `libgtk-3-dev libwebkit2gtk-4.1-dev` (Wails 2.10 on ubuntu-latest/24.04).
- Commits are GPG-signed; if signing hangs, retry once (gpg-agent re-warm), then report — never `--no-gpg-sign`.

- [ ] **Step 1: Create the allowlist** — `.github/govulncheck-allow.txt`

```
# Accepted/deferred govulncheck IDs. The gate (scripts/govulncheck-gate.sh)
# fails on any code-affecting vuln NOT listed here.
#
# These 5 are go-ethereum (github.com/ethereum/go-ethereum) devp2p/RLPx p2p
# vulns (DoS / crash / handshake key-validation). They are reachable ONLY via
# the opt-in embedded node (internal/embeddednode runs go-ethereum's p2p stack);
# the default remote/local node modes are pure WebSocket RPC clients with no p2p
# exposure. None touch keys/signing/funds. They cannot be bumped without
# breaking the pinned go-zenon build (verified vs go-ethereum v1.16.8/1.16.9/
# v1.17.0), and no newer go-zenon exists. Deferred pending go-zenon's upstream
# libp2p migration, which removes this stack. Revisit then.
GO-2026-4314
GO-2026-4315
GO-2026-4507
GO-2026-4508
GO-2026-4511
```

- [ ] **Step 2: Create the gate script** — `scripts/govulncheck-gate.sh`

```bash
#!/usr/bin/env bash
# Fail if govulncheck reports any code-affecting vulnerability whose ID is not
# in .github/govulncheck-allow.txt. govulncheck has no native ignore, so we diff
# the reported IDs against the allowlist.
set -uo pipefail

allow_file="$(dirname "$0")/../.github/govulncheck-allow.txt"
allow="$(grep -oE 'GO-[0-9]+-[0-9]+' "$allow_file" | sort -u)"
found="$(govulncheck ./... 2>/dev/null | grep -oE 'GO-[0-9]+-[0-9]+' | sort -u)"

unexpected="$(comm -23 <(printf '%s\n' "$found") <(printf '%s\n' "$allow"))"

if [ -n "$unexpected" ]; then
  echo "FAIL: govulncheck reported non-allowlisted vulnerabilities:"
  printf '%s\n' "$unexpected"
  echo "If a finding is genuinely accepted, add it to .github/govulncheck-allow.txt WITH justification."
  exit 1
fi

echo "OK: only allowlisted vulnerabilities present:"
printf '%s\n' "$found"
```

Then: `chmod +x scripts/govulncheck-gate.sh`.

## Task 4: The CI workflow

**Files:** Create `.github/workflows/ci.yml`.

**Interfaces:**
- Consumes: Task 2's `.gosec.json`, Task 3's `scripts/govulncheck-gate.sh`, Task 1's `go.mod` (go 1.25.11).
- Produces: the PR gate workflow.

- [ ] **Step 1: Write the workflow** — `.github/workflows/ci.yml`

```yaml
name: CI

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  frontend:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: frontend
    steps:
      - uses: actions/checkout@v4
      - run: corepack enable
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
          cache-dependency-path: frontend/pnpm-lock.yaml
      - run: pnpm install --frozen-lockfile
      - run: pnpm run check
      - run: pnpm test

  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
      - name: govulncheck (allowlist gate)
        run: |
          go install golang.org/x/vuln/cmd/govulncheck@latest
          bash scripts/govulncheck-gate.sh
      - name: gosec
        run: |
          go install github.com/securego/gosec/v2/cmd/gosec@latest
          gosec -conf .gosec.json ./...

  build-test:
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
      - run: corepack enable
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
          cache-dependency-path: frontend/pnpm-lock.yaml
      - name: Linux WebKit deps
        if: runner.os == 'Linux'
        run: sudo apt-get update && sudo apt-get install -y libgtk-3-dev libwebkit2gtk-4.1-dev
      - run: go vet ./...
      - run: go test ./...
      - name: Install Wails
        run: go install github.com/wailsapp/wails/v2/cmd/wails@v2.10.1
      - name: Wails build
        run: wails build
```

- [ ] **Step 2: Validate the YAML parses**
